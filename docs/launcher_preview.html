流水线说明

YOLO 输出按顺序流经：删除 → 过滤 → 瞄准。
• 删除：该类别的检测框直接丢弃，不再进入任何下游。
• 过滤：保留检测框（调试视图与深度遮罩可见），但不会作为瞄准候选。
• 瞄准：作为瞄准候选；优先级由"瞄准热键"面板决定。

当前 UI 要点
• 左栏里少掉的：按键绑定（统一到"瞄准热键"）、游戏内叠层（整个模块已移除）；多出来的：瞄准热键 与 硬件。
• 没有任何运行时硬编码热键：F2 / F3 / F4 / HOME 等旧功能键全部移除；设置窗口在整个会话期间一直可见，无显隐切换。
• 侧栏分组按 kOverlayTabs 顺序渲染：核心 与 控制 会按 tab 顺序重复出现（例如 核心 → 控制 → 核心），不是 bug，是当前源码的排列方式。
• 目标面板：类别 以"删除 / 过滤 / 瞄准"三栏单选；"删除"的类别在 NMS 后立即丢弃，"过滤"仍可被调试视图看到但不参与瞄准。
• 瞄准热键：每个热键持有完整一套 FOV / PID（含吸附锁死力度）/ Kalman 参数；未按下任何热键时鼠标不动。原来的 Snap / Near 独立面板已合并，不再有单独的"目标修正"子段。
• 鼠标 面板已移除，所有参数并入每个热键自身。
• 检测预览 现在只在「启动器」面板提供；性能统计面板已移除该子段，只保留耗时分解、采集 FPS 与采集详情。
• AI 模型面板包含 深度推理 / 深度遮罩 子段（仅在 TRT 后端可用）。
• 界面外观不再有防截屏。
